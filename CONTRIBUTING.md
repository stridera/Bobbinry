# Contributing to Bobbinry

Thanks for your interest!

## How to contribute
1. **Discuss first:** Open an issue describing the problem or proposal.
2. **Fork & branch:** `feat/xyz` or `fix/abc` naming.
3. **Run checks:** `pnpm i && pnpm -w dlx turbo run typecheck lint build`.
4. **Tests:** Add or update tests for changes.
5. **PR:** Link issues, explain context, include screenshots for UI changes.

## Guidelines
- Keep PRs focused and reasonably sized.
- Document user-facing changes in `README` or `/docs`.
- Security-relevant changes must include a rationale and threat model note.
